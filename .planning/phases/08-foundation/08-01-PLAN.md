<task type="auto">
  <name>Task 1: Add FOUC blocking script to index.html</name>
  <files>index.html</files>
  <action>
Add an inline blocking `<script>` as the FIRST element inside `<head>` after `<meta charset="UTF-8" />`. This script must appear BEFORE the `<meta name="viewport">`, `<link rel="preconnect">`, and Google Fonts `<link>` tags.

Script logic:
1. Read `localStorage.getItem('cl-theme')` (key is `cl-theme` to avoid collisions)
2. If stored value is `'light'` or `'dark'`, apply it directly
3. Otherwise (null, `'auto'`, or any other value), detect system preference via `window.matchMedia('(prefers-color-scheme: dark)').matches`
4. Set `document.documentElement.setAttribute('data-theme', resolved)`
5. Wrap in try/catch for localStorage access errors (private browsing)

Exact script (use this verbatim):
```html
<script>
  (function() {
    try {
      var stored = localStorage.getItem('cl-theme');
      var system = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
      document.documentElement.setAttribute('data-theme',
        stored === 'light' || stored === 'dark' ? stored : system
      );
    } catch(e) {}
  })();
</script>
```

Also update `<meta name="theme-color" content="#060606">` to `<meta name="theme-color" content="#0F1419">` (new dark mode base color).

Do NOT modify the `<body>` or any other element. Do NOT move or remove existing `<link>` or `<meta>` tags — only add the script before them and update the theme-color meta.
  </action>


WCAG AA compliance notes for light palette:
- `#555555` on `#FFFFFF` = 7.4:1 (passes AA for body text)
- `#777777` on `#FFFFFF` = 4.48:1 (passes AA for body text at 4.5:1 threshold)
- `#1A1A1A` on `#F8F7F4` = 16:1+ (passes AAA)
- Gold `#C9A84C` on light backgrounds: 2.5:1 (FAILS for text) — gold remains decorative only in light mode
