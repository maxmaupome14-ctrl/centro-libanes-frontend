**File 1: `src/store/themeStore.ts`**

Zustand store managing theme mode and resolved theme. Key details:
- Types: `ThemeMode = 'light' | 'dark' | 'auto'`, `ResolvedTheme = 'light' | 'dark'`
- Storage key: `'cl-theme'` (MUST match the FOUC blocking script in index.html)
- On init: read `localStorage.getItem('cl-theme')`, default to `'auto'` if not found
- `resolveTheme(mode)`: if `auto`, use `window.matchMedia('(prefers-color-scheme: dark)').matches`; otherwise return mode directly
- `applyTheme(resolved)`: `document.documentElement.setAttribute('data-theme', resolved)`
- `setMode(mode)` action: (1) persist mode to localStorage with key `'cl-theme'`, (2) compute resolved, (3) call `applyTheme`, (4) if mode is `'auto'` subscribe to `matchMedia('(prefers-color-scheme: dark)').addEventListener('change', ...)`, if mode is NOT `'auto'` remove any existing listener
- Store the `MediaQueryList` reference and cleanup function at module level to prevent listener leaks (Pitfall 4 from research)
- On store creation: immediately resolve and apply theme (so React and the FOUC script stay in sync)

**File 2: `src/store/platformStore.ts`**

Module-level constant, NOT a Zustand store (no state changes ever). Key details:
- Type: `Platform = 'ios' | 'android' | 'web'`
- `detectPlatform()` function:
  1. Try `Capacitor.getPlatform()` from `@capacitor/core` — if result is `'ios'` or `'android'`, return it
  2. UA fallback: test `/iPad|iPhone|iPod/` for `'ios'`, `/Android/` for `'android'`
  3. Default: return `'web'` (maps to iOS visual treatment per PLAT-06)
- Export `const PLATFORM: Platform = detectPlatform()` at module level — runs once, cached forever
- Export the `Platform` type

**File 3: `src/hooks/useTheme.ts`**

Simple hook wrapping themeStore:
```typescript
export function useTheme() {
  const mode = useThemeStore((s) => s.mode);
  const resolved = useThemeStore((s) => s.resolved);
  const setMode = useThemeStore((s) => s.setMode);
  return { mode, resolved, setMode };
}
```

**File 4: `src/hooks/usePlatform.ts`**

Hook returning platform info (no state, pure computation from module-level constant):
```typescript
export interface PlatformInfo {
  platform: Platform;
  isIOS: boolean;
  isAndroid: boolean;
  isWeb: boolean;
}

export function usePlatform(): PlatformInfo {
  return {
    platform: PLATFORM,
    isIOS: PLATFORM !== 'android',  // web + ios both use iOS style (PLAT-06)
    isAndroid: PLATFORM === 'android',
    isWeb: PLATFORM === 'web',
  };
}
```

**File 5: `src/components/providers/AppProviders.tsx`**

Thin wrapper that:
1. Reads `resolved` from `useThemeStore`
2. On mount: sets `document.documentElement.setAttribute('data-platform', PLATFORM)` once
3. On `resolved` change: sets `document.documentElement.setAttribute('data-theme', resolved)` — this keeps React in sync with the FOUC script's initial value
4. Renders `{children}` with no extra DOM

```typescript
import { useEffect } from 'react';
import { PLATFORM } from '../../store/platformStore';
import { useThemeStore } from '../../store/themeStore';

export function AppProviders({ children }: { children: React.ReactNode }) {
  const resolved = useThemeStore((s) => s.resolved);

  useEffect(() => {
    document.documentElement.setAttribute('data-platform', PLATFORM);
  }, []);

  useEffect(() => {
    document.documentElement.setAttribute('data-theme', resolved);
  }, [resolved]);

  return <>{children}</>;
}
```
